# C4

Dynamic architecture visualization for cloud-native systems. Connect your Azure environment, auto-discover resources, and get interactive C4-model diagrams with real-time telemetry overlays and AI-powered analysis.

## Features

- **Auto-discovery** -- Connects to Azure subscriptions and maps resources, relationships, and topology via Azure Resource Graph.
- **C4 model diagrams** -- Interactive context, container, and component-level views with drill-down navigation.
- **Live telemetry** -- Overlays Application Insights metrics. Connections are color-coded green/yellow/red by health score.
- **IaC drift detection** -- Compares Bicep/Terraform desired state against live Azure state. Discrepancies highlighted on the diagram.
- **AI analysis** -- Architecture review and STRIDE-based threat assessment via Semantic Kernel with local Ollama models.
- **Export** -- Download diagrams as SVG or PDF.

## Quick start

```bash
cp .env.example .env
docker compose up
```

Open [localhost:3000](http://localhost:3000). API at [localhost:5000](http://localhost:5000), Swagger at [localhost:5000/swagger](http://localhost:5000/swagger).

# Frontend
cd web && npm install && npm run dev
```

### Tests

```

## Architecture

Modular monolith -- .NET 9 backend, React 19 frontend. Five bounded-context modules communicate through explicit contracts, each following Vertical Slice Architecture with Ports and Adapters.

```
src/
  Host/                  ASP.NET Core composition root
  Modules/
    Identity/            Organizations, projects, members, RBAC
    Discovery/           Azure subscription connection, resource discovery, drift
    Graph/               C4-model graph construction and versioning
    Telemetry/           Application Insights ingestion and health scoring
    Visualization/       Diagram rendering, export, real-time updates (SignalR)
  Shared/
    Kernel/              Result<T>, Entity, AggregateRoot, strongly-typed IDs
    Infrastructure/      EF Core base, MediatR pipeline, validation behavior
web/                     React + TypeScript + React Flow
infra/                   Terraform (Scaleway) + cloud-init
docs/
  architecture/          ADRs and architecture guides
  standards/             Coding and testing standards
```

## Deployment

Terraform configuration for a Scaleway DEV1-S instance (~3 EUR/month). GitHub Actions handles the full pipeline.

```bash
cd infra && terraform init && terraform apply
```

The CI/CD workflow (`.github/workflows/deploy.yml`) runs on push to `main`:

1. **Test** -- `dotnet test` + `npm test`
2. **Build** -- Docker images pushed to GitHub Container Registry
3. **Provision** -- `terraform apply` (state stored on `terraform-state` branch)
4. **Deploy** -- SSH, pull images, `docker compose up`

Required repository secrets:

| Secret | Description |
|--------|-------------|
| `SCW_ACCESS_KEY` | Scaleway API access key |
| `SCW_SECRET` | Scaleway API secret key |
| `SCW_ORGANIZATION_ID` | Scaleway organization ID |
| `SCW_PROJECT_ID` | Scaleway project ID |

SSH keys are auto-generated by Terraform and registered with Scaleway IAM -- no manual key management needed.
